# 更新日志 (Changelog)

所有本项目的重大更新、重构以及功能改动均记录在此文件中。

## [v1.2.0] - 2026-05-29

### 🚀 新增功能
* **PVE 宿主指标 exporter 导出器**：编写并在 PVE 主机 (192.168.31.200) 部署了独立、零依赖的 python metrics 服务 `pve-metrics.service`，监听 9101 端口，安全输出 CPU Package、核心温度以及宿主系统盘 SMART 温度 JSON。
* **物理设备免密 SMART 采集**：
  - 自动识别飞牛上的 1TB 移动硬盘为 `/dev/sdb`（PVE 系统盘为 `/dev/sda`），确保盘符无误。
  - 为飞牛 `smartctl` 配置了独立的安全免密规则 `/etc/sudoers.d/kindle-panel-smartctl`，实现普通用户 `rvc` 零密码读取 SMART 温度。
* **1s 秒级高频硬件温控更新**：
  - 支持将后端硬件温度轮询率提升到 `1` 秒以实现极其灵敏的监控。
  - 前端 AJAX 定时刷新率由 2s 改为 `1` 秒 (`1000`ms)。
  - AJAX 接口 `/api/status` 与前端同步扩展 `hw_rows` 并且在 `with lock:` 外部安全渲染规避了非重入锁死锁死循环风险。

### 🎨 UI 与双栏对称美学优化
* **右栏卡片流式垂直布局**：
  - 将“主机状态”卡片（CPU 温度、系统盘、USB 硬盘）移至右列“飞牛 OS 运行状态”卡片的正下方，左侧完整呈现“未来 7 天”大卡片。
  - 左右两列完美对齐，完美拉匀两边行数，继续维持 Kindle E-ink 浏览器高兼容的纯 Table 工业极简对称美学。
* **页脚双时间戳动态拼接**：页脚展示更新为 `数据更新 天气 HH:MM · 主机 HH:MM`，AJAX 刷新实时拼接天气与硬件更新时刻。

## [v1.1.0] - 2026-05-25

### 🚀 新增功能
* **飞牛 OS 运行状态实时采集**：纯 Python 标准库实现对 Linux 宿主机 `/proc` 系统的零依赖解析：
  * **运行时间**：读取 `/proc/uptime` 并转换成“X天 Y小时 Z分”人性化时间。
  * **CPU 利用率**：读取 `/proc/stat` delta 计算实时多核 CPU 占比。
  * **内存已用大小**：读取 `/proc/meminfo` 的已用与可用物理内存空间。
  * **网络上行/下行流速**：计算物理网卡（过滤掉虚拟网卡与 Docker 桥接网卡）流量 delta。
  * **硬盘读写 IO 速度**：计算真实块设备磁盘 `/proc/diskstats` 扇区读取与写入速度。
* **高频秒级轮询引擎**：将飞牛 OS 本机性能指标采集集成进后台秒级高频更新线程 `local_status_refresh_loop()`，实现 1s 级别数据捕获，无需额外开辟线程，资源占用极低。

### 🎨 UI 与双栏对称美学优化
* **左天气·右飞牛双栏对齐排版**：设计并实施了旧款 Kindle 绝对兼容的 `width: 100%` 双栏 `<table>` 布局（49% 天气 : 2% 间隔 : 49% 飞牛），整体页面边界无任何溢出或拓展。
* **对称高度拉平**：右侧飞牛性能监控精简为 7 行核心指标，左侧天气预报对应压缩风速风向为 7 行（日期、天气文字、最低/最高气温），使得两边高度绝对一致，在墨水屏上展现出极致的对称工业设计美。

## [v1.0.0] - 2026-05-25

### 🚀 新增功能
* **极简主义天气日历面板**：正式发布基于 HTML/CSS Table 对齐机制的天气日历看板，集成了实时天气气温、体感、湿度、风力、空气质量 (AQI)、未来 7 天预报、本月日历（当天黑底白字高亮）以及大字号实时服务器时间。
* **Open-Meteo 天气引擎接入**：全线切换至 Open-Meteo 免 Key 公益天气服务，实现零配置、零鉴权、完全免费的后端天气体系，无缝适配合肥包河区。
* **IPv4-Only 强制解析补丁**：内置 4 行 Socket 猴子补丁，强制 Python 仅执行 IPv4 域名解析，成功攻克由于飞牛 OS 宿主机破损的上游 IPv6 DNS 导致的 `read operation timed out` 核心网络超时故障。
* **systemd 服务托管**：提供 `kindle-panel.service` 系统单元配置文件，支持 FlyOS 系统重启自拉起和异常中断进程自愈。
* **免密静默代理执行 (run.sh)**：内置 IDE 自动代理机制，避免开发调试中频繁的人工弹窗确认，实现全自动化静默执行与维护。

### 🎨 UI 与设计优化
* **对齐工整的 HTML 结构**：重构前台 HTML 布局，弃用 Flexbox 这种旧版 Kindle 浏览器极易渲染失败的现代属性，采用 100% 物理对齐的等宽 `<table>` 表格布局。
* **E-Ink 墨水屏消除模糊**：在 CSS 中注入 `-webkit-font-smoothing: none;`，彻底关停抗锯齿，使边缘在 Kindle 电子墨水屏上保持极度锐利，拒绝字体晕染模糊。
* **前端主动轮询刷新**：加入 `<meta http-equiv="refresh" content="300">`，每 5 分钟自动发起前台重绘轮询。
